# HTTP几种请求方法和区别

## HTTP1.0

- GET：指定页面信息，返回实体主体
- POST：项指定资源提交数据进行处理请求。数据被包含在请求体中。POST请求可能会导致新的资源建立和/或已有资源的修改
- HEAD：类似于GET请求，返回的相应中没有具体的内容，用于获取请求头

## HTTP1.1

- OPTIONS：允许客户端查看服务器性能（服务器能够支持那些请求）
- PUT：上传文档内容（能够**指定**资源）
- DELETE：删除服务器资源
- TRACE：跟踪。用于测试或诊断
- CONNECT：把请求连接转换到透明的`TCP/IP`通道
- PATCH：对PUT的补充，对已知资源进行局部更新
